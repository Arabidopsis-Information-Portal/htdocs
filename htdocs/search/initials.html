Using the TAIR Search Page

The TAIR Search Page gives you access to the three main tasks you can perform with the TAIR database:

  • Searching
    You can do simple or complex searching. Simple searching is a name/keyword search of all data types or a specific data type. Complex searching uses additional parameters to search all data types or a specific data type.
    In the prototype version, all seraching is available for Clone and Marker data.
  • Browsing
    You can browse specific types of data organized by type, position, or alphabetically. All entries in the browsable pages are linked to detailed information. The browsable pages will be updated regularly to contain the most up-to-date information from the database.
  • Submitting data
    You can submit and update data in the database using the web data submission option and forms. All user-submitted data are checked for consistency in a semi-automated way and users are notified by email of any inconsistencies.

General Search

At the top of the TAIR Search Page, are the General Search options.

General Search lets you search all data TAIR by name or keyword. This search is particularly useful if you have the name of an entity but do not know the data class to which it belongs. TAIR's database contains many types of data, which are defined and described in detail in the TAIR Data Model and Documentation.

Using Wildcards

You can use wildcard characters in your name searches. Four wildcard characters/combinations are supported:
*

Matches 0 or more characters

?

Matches exactly one character

\*

Matches the character "*"

\?

Matches the character "?"

Specific Search

The Specific Search options appear below the General Search options.

These options let you search within specific data types in TAIR. The data types (described below) are listed on the left. Search, browse, and submit data options appear to the right of each data type. When you select the search option for a data type, a search form appears, with options for both simple and complex searching. The specific search is most useful if you know the type of data you seek.

Data Types

Community

This data type includes the organizations and individual researchers who study Arabidopsis. Organizations may include Arabidopsis labs, stock centers, committees, facilities, universities, and non- and for-profit institutes.

Reference

This data type represents reference information for Arabidopsis data. The types of reference in TAIR include journal article, book, book chapter, thesis, personal communication, electronic references, conference proceedings, analysis reference (for software-analyzed data such as gene annotation), database reference (data from external databases such as GenBank and TIGR), and websites.

Maps

This data type represents any entity that has other entities assigned to (or positioned on) it. Maps are largely separated into genetic, physical, and sequence maps in TAIR. However, entites such as clones and genes will also be considered to be maps in TAIR, if the entities have other elements assigned to them.

Markers

This data type represents genetic markers (phenotypic and molecular) and ecotype-specific polymorphisms in Arabidopsis. Also, potential polymorphisms (e.g., dinucleotide repeats or satellites) are included in this category for searching.

Clones

This data type represents any piece of DNA that is inserted into a vector.

Genes

The genes data type represents any Arabidopsis gene. A gene in TAIR includes the coding and transcribed regions and all of the sequences (introns, exons, cis-3' and 5' regulatory regions) required for expression. A gene can have one to many associated sequences and one to many gene products. A gene can be experimentally determined or predicted by sequence analysis (genome annotation).

Sequences

This data type contains sequence text and information for all Arabidopsis nucleotide and amino acid sequences. Nucleotide sequence types include genomic, cDNA, oligos, polymorphic sequences, and sequences flanking molecular objects such as genetic markers (e.g., primers).
